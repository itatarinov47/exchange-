# Обменник Нячанг — Telegram Mini App

Сервис для обмена валюты через Telegram: клиент открывает мини-приложение,
видит курс, рассчитывает сумму, выбирает встречу или доставку — заявка
сразу приходит администратору в Telegram с кнопками «Подтвердить» / «Отклонить».

## Что внутри

- `server.js` — веб-сервер (Express): API курсов и заявок, отдаёт мини-апп
- `bot.js` — Telegram-бот (grammY): `/start`, админ-команды, уведомления
- `public/` — само мини-приложение (HTML/CSS/JS)
- `data/rates.json` — курсы валют (редактируются командой `/setrate` или вручную)
- `data/meeting-points.json` — точки встречи (замените на свои реальные адреса)
- `data/orders.json` — заявки клиентов (создаётся автоматически)

Хранение — простые JSON-файлы. Этого достаточно для старта; если поток заявок
вырастет, стоит перейти на настоящую БД (Postgres/SQLite) — дайте знать, помогу перенести.

## 1. Создать бота

1. Откройте @BotFather в Telegram → `/newbot` → задайте имя и юзернейм.
2. Сохраните выданный токен — это `BOT_TOKEN`.
3. Узнайте свой Telegram ID через @userinfobot — это `ADMIN_CHAT_ID`
   (сюда будут приходить уведомления о заявках).

## 2. Настроить окружение

```bash
cp .env.example .env
```

Заполните `.env`:

```
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_CHAT_ID=ваш_telegram_id
WEBAPP_URL=https://ваш-домен.com
PORT=3000
```

`WEBAPP_URL` обязательно должен быть **HTTPS** — Telegram не откроет мини-апп по http.

## 3. Установить зависимости и запустить

```bash
npm install
npm start
```

Сервер поднимется на указанном порту и одновременно запустит бота.

## 4. Деплой (чтобы WEBAPP_URL был настоящим HTTPS-адресом)

Проще всего — один из бесплатных/дешёвых хостингов с авто-HTTPS:

- **Railway** или **Render**: подключаете репозиторий, задаёте переменные окружения
  из `.env`, деплой запускает `npm start` автоматически.
- Для теста на своём компьютере без хостинга — можно временно поднять туннель
  (`ngrok http 3000`) и указать выданный ngrok-адрес как `WEBAPP_URL`.

После деплоя зайдите в @BotFather → `/mybots` → выберите бота → **Bot Settings →
Menu Button** → укажите тот же `WEBAPP_URL`, чтобы кнопка открытия мини-аппа
была доступна и из меню чата.

## 5. Команды администратора в боте

- `/rates` — показать текущие курсы
- `/setrate USD 25300 25650` — задать курс покупки/продажи для валюты
- `/pending` — показать все заявки в ожидании с кнопками подтверждения

## 6. Что нужно донастроить под себя

- **Точки встречи**: отредактируйте `data/meeting-points.json` — впишите реальные
  адреса/ориентиры в Нячанге, где вы готовы встречаться с клиентами.
- **Курсы**: задайте актуальные через `/setrate` или прямо в `data/rates.json`.
  `buy` — курс, по которому вы покупаете валюту у клиента (он отдаёт USD, получает VND);
  `sell` — по которому продаёте (клиент отдаёт VND, получает USD).
- **Способ доставки курьером**: сейчас это просто текстовое поле «адрес + время» —
  логистику (кто едет, маршрут) ведёте вручную через уведомления в боте. Если нужно
  закрепление курьеров за заявками — это следующий шаг развития.
- **Несколько админов**: пока поддержан один `ADMIN_CHAT_ID`. Если нужна команда —
  легко расширить на список ID.

## Ограничения текущей версии (MVP)

- Обмен между двумя иностранными валютами (например USD → EUR) не поддержан —
  только VND ↔ иностранная валюта. Это покрывает 95% реальных запросов в Нячанге.
- Платёжный приём (онлайн-оплата) не подключён — расчёт происходит при встрече/доставке,
  как и было задумано.
- Хранение в JSON подходит для одного обменника с умеренным потоком заявок;
  при росте — миграция на БД.
